# Configuration

このドキュメントでは、`doc-repo.config.json`の設定項目について説明します。

## 設定ファイルの初期化

```bash
npm run doc-repo -- init
```

カレントディレクトリに`doc-repo.config.json`が作成されます。

```json
{
  "name": "Doc Repo",
  "rootDir": ".",
  "include": ["**/*.md"],
  "exclude": [],
  "port": 4000
}
```

設定ファイルがすでに存在する場合は上書きされません。

## 設定項目

| 項目      | 型         | デフォルト                     | 説明                                                     |
| --------- | ---------- | ------------------------------ | -------------------------------------------------------- |
| `name`    | `string`   | `"Doc Repo"`                   | Viewerのサイドバーに表示する名前                         |
| `rootDir` | `string`   | 設定ファイルのあるディレクトリ | Markdownファイルの収集と保存に使用するルートディレクトリ |
| `include` | `string[]` | `["**/*.md"]`                  | 対象とするMarkdownファイルのglobパターン                 |
| `exclude` | `string[]` | `[]`                           | 追加で除外するファイルのglobパターン                     |
| `port`    | `number`   | `4000`                         | ワークスペースのHTTPサーバーポート                       |

すべての項目は省略できます。

- `rootDir`の相対パスは、設定ファイルがあるディレクトリを基準に解決されます。
- `include`に空配列を指定した場合は、デフォルトの`["**/*.md"]`が使用されます。
- `exclude`は`include`より優先されます。
- `node_modules/**`、`.git/**`、`.doc-repo/**`は常に除外されます。
- コマンドラインオプションの`--port`は、設定ファイルの`port`より優先されます。
- `port`には`1`から`65535`までの整数を指定できます。

## 設定ファイルの探索

doc-repoは、カレントディレクトリから上位へ`doc-repo.config.json`を探索し、最初に見つかったファイルを使用します。

設定ファイルが見つからない場合は、Gitリポジトリのルートを使用します。Gitルートも見つからない場合は、カレントディレクトリを使用します。

## 設定エラー

設定ファイルを読み込めない場合や、設定値の型・範囲が正しくない場合、doc-repoはエラーを表示して終了します。
