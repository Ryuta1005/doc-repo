# Contract: Viewer Sidebar Create UX（Story 022）

## Scope

サイドバー上での新規文書作成 UX 契約を定義する。

## Hover Action Visibility

- ファイルまたはフォルダ行をホバーしている間のみ、行右側に `+` ボタンを表示する
- 同時に複数の `+` を表示しない
- ホバー解除時は `+` を非表示に戻す

## Creation Flow Entry

- `+` クリックで編集画面へ直接遷移する（作成モーダルは使わない）
- 遷移時に `CreationAnchorContext` を保持する
  - folder 起点: そのフォルダ直下
  - file 起点: 親フォルダ直下（同階層）

## Inline Filename Field Contract

- 編集画面で、ツールバーと本文入力欄の間にファイル名入力欄を表示する
- 入力欄は下線のみを表示し、空欄時はグレーの `ページタイトル` を表示する
- `+` 押下直後はまだ物理ファイルを作成しない

## Input Rules in Create Screen

- 入力欄はファイル名のみを受け付ける
- `/` `\\` を含む入力は拒否し、理由を表示する
- 入力値全体をファイル名本文として扱い、末尾に常に `.md` を自動付与
- `.md` を含む入力もファイル名本文として扱う

## Success UX Contract

- 保存ボタン押下時に初めて新規ファイル作成を行う
- 成功後、文書ツリーを更新する
- 作成文書を選択状態にする
- サイドバー上の Markdown 文書表示名は、既存文書と新規作成文書の両方で最後の `.md` を外した名前とする
- 作成文書の初期本文は空本文とする

## Failure UX Contract

- 作成失敗時は理由を表示する
- 失敗理由は再試行可否の判断に使える内容とする
- 既存同名ファイルがある場合は上書きせず失敗を表示する

## Unsaved Changes Guard

- 未保存変更がある場合、既存の確認フローを作成操作にも適用する
- 利用者の確認結果に応じて、作成フロー継続または中止を行う

## Explicit Out of Scope

- サイドバー外からの新規作成導線
- フォルダ作成
- 削除操作

## Verification Checklist

- ホバー時のみ `+` が表示される
- `+` クリックで編集画面（create draft）に遷移する
- file/folder で作成先解決が仕様通りに分岐する
- `+` 押下時点ではファイルが作成されず、保存時に作成される
- 入力値へ常に `.md` が付与されて作成できる
- サイドバー表示が最後の `.md` なしで統一される
- 未保存変更ありで既存ガードが機能する
