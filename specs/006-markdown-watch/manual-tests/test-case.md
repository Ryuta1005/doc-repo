## MT-001 change 保存時の自動再生成と自動反映確認

目的:

- `change` 検知後に再生成され、成功時のみブラウザが自動更新されることを確認する。

手順:

1. `node dist/cli/index.js serve`
2. ブラウザで表示 URL を開く。
3. 既存の `.md` を編集して保存する。
4. 標準出力の `change detected` -> `regenerate started` -> `regenerate succeeded` を確認する。
5. ブラウザが手動リロードなしで更新されることを確認する。

期待結果:

- 変更検知から更新反映まで 5 秒以内
- 成功時にのみ reload が配信される

## MT-004 debounce + pending + 同時実行禁止確認

目的:

- 300ms debounce、pending 再実行、同時再生成なしを確認する。

手順:

1. `node dist/cli/index.js serve`
2. 同一 `.md` を短時間で連続保存する（3回以上）。
3. ログで再生成開始回数が過剰でないことを確認する。
4. 再生成中に追加保存を1回行う。
5. 現在処理完了後にもう1回だけ再生成されることを確認する。

期待結果:

- 最後の変更から約 300ms 後に再生成される
- 再生成は同時に複数走らない
- 再生成中の変更は破棄されず、完了後に1回反映される

## MT-005 再生成失敗時 no-reload 確認

手順:

6. テスト後 `mv templates.__bak__ templates` で復元する。

期待結果:

- 失敗時に `reload` は送信されない
