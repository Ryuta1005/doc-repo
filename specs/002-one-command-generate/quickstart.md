# Quickstart: ワンコマンド生成

## 1. 前提

- Node.js LTS が利用可能
- 対象ディレクトリに Markdown ファイルが存在（0件でも動作確認は可能）

## 2. 実行

```bash
# 対象リポジトリ直下で実行
node dist/cli/index.js
```

## 3. 期待結果

- `.doc-repo` ディレクトリが生成される
- 実行コンソールに成功メッセージと出力先が表示される
- Markdown 0件の場合は、成功メッセージに加えて警告が表示される

## 4. 再実行確認

```bash
# Markdown を編集後に再実行
node dist/cli/index.js
```

- `.doc-repo` は全置換で更新される
- 古い残骸ファイルが残らない

## 5. 失敗時確認

- 入力条件を壊して実行した場合、終了コードが 0 以外になる
- 失敗理由と対処の方向性がコンソールに表示される

## 6. 終了コード確認（任意）

```bash
node dist/cli/index.js
echo $?
```

- 成功（警告付き成功を含む）: `0`
- 失敗: `0` 以外

## 7. 検証ログ（2026-06-07）

- 成功ケース: `.doc-repo/index.html` と `.doc-repo/content.json` の生成を確認
- 警告付き成功ケース: 非Git一時ディレクトリで Markdown 0 件時に警告表示と exit code `0` を確認
- 失敗ケース: `templates/` 不在時に失敗メッセージと exit code `1` を確認
- 再実行ケース: 既存マーカーを置いた後の再実行で全置換されることを確認
- 失敗時保持: 既存 `.doc-repo` がある状態で失敗しても既存成果物が保持されることを確認
