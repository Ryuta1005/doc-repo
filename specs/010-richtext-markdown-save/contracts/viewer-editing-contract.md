# Contract: Viewer Editing UX（Story 010）

## Scope

閲覧画面から編集し保存するまでの UI 契約を定義する。

## Mode Transition

- 閲覧モードから編集モードへ切り替え可能
- 編集開始時に現在の Markdown 本文が編集画面へロードされる
- 保存成功後は閲覧モードへ戻り、最新表示が確認できる
- 破棄操作で保存せず閲覧モードへ戻れる

## Supported Editing Features

- 本文
- 見出し 1 / 2 / 3
- 太字
- イタリック
- Markdown 入力ルール
  - `# `, `## `, `### `
  - 太字、イタリックの入力ルール

下線は本 Story の対象外。

## Unsaved Changes Guard

未保存変更がある場合、以下トリガーで確認ダイアログを表示する。

- 別ファイル選択
- 編集終了
- ブラウザ離脱

利用者は以下を選べる。

- 編集継続
- 変更破棄

## Warning Flow for Unsupported Segments

- 保存前に未対応要素の存在を警告する
- 利用者は保存継続またはキャンセルを選択できる
- 保存継続時は原文パススルーを優先して保存する

## Save Result Feedback

利用者に次を明示する。

- 保存成功
- 保存失敗（3分類）
  - 保存対象不正
  - 対象文書を保存不能
  - 一時的な保存失敗

各失敗で理由と再試行可否を表示する。

## Explicit Out of Scope

- 外部変更の競合検知
- 版情報照合
- 競合時の保存拒否ポリシー

これらは Story 011 で扱う。
