## User Scenarios & Testing _(mandatory)_

### User Story 1 - 閲覧中文書を編集して保存する (Priority: P1)

利用者は閲覧画面から編集モードへ切り替え、本文・見出し1〜3・太字・イタリックを使って文書を修正し、元の Markdown 文書として保存できる。

**Why this priority**: この機能がないと「Markdown 記法を知らない利用者でも文書を更新できる」という中核価値が成立しないため。

**Independent Test**: 既存 Markdown 文書を開き、編集モードでテキストと書式を変更して保存し、再表示で変更が反映されることを確認すれば、この価値を単独で検証できる。

**Acceptance Scenarios**:

1. **Given** 利用者が表示中の Markdown 文書を開いている, **When** 編集モードへ切り替える, **Then** 現在の本文内容が編集画面に表示される
2. **Given** 編集画面で見出し1〜3、太字、イタリックを設定した, **When** 保存する, **Then** 書式の意味を維持した Markdown として元文書へ保存される
3. **Given** 保存が成功した, **When** 閲覧画面へ戻る, **Then** 生成済み表示が自動更新され最新内容が確認できる

---

### User Story 2 - 誤操作による変更消失を防ぐ (Priority: P2)

利用者は編集中に誤ってファイル切替・編集終了・離脱を行っても、未保存変更の確認を通じて変更消失を防げる。

**Why this priority**: 保存前の離脱で内容が失われると実運用での信頼性が大きく下がるため。

**Independent Test**: 編集内容を作成した状態で、別ファイル選択・編集キャンセル・ページ離脱を試し、確認が表示されることと、破棄/継続を選べることを確認すれば独立検証できる。

**Acceptance Scenarios**:

1. **Given** 未保存変更がある, **When** 別ファイルを選択する, **Then** 変更破棄確認が表示され、利用者が継続編集または破棄を選べる
2. **Given** 未保存変更がある, **When** 編集終了またはブラウザ離脱を試みる, **Then** 変更破棄確認が表示される
3. **Given** 利用者が変更破棄を選択した, **When** 閲覧画面へ戻る, **Then** 編集前の内容が保持される

---

### User Story 3 - 保存リスクを事前に判断して操作する (Priority: P3)

利用者は未対応要素を含む文書を保存する前に警告を確認し、保存継続またはキャンセルを選べる。さらに保存対象外パスへの保存要求は拒否され、誤保存や不正保存を防げる。

**Why this priority**: 保存時の内容欠損リスクと不正な保存先指定を利用者が認識できることが、安全に利用する前提になるため。

**Independent Test**: 未対応要素を含む文書で保存操作時の警告表示とキャンセル可否を確認し、許可外ファイルや不正パスへの保存が拒否されることを確認すれば独立検証できる。

**Acceptance Scenarios**:

1. **Given** 保存時に未対応要素が検出される, **When** 利用者が保存操作を行う, **Then** 変換または欠損の可能性を明示した警告が表示され、保存継続またはキャンセルを選べる
2. **Given** 保存対象がルート外・非 Markdown・除外対象のいずれかに該当する, **When** 保存を実行する, **Then** 保存は拒否され理由が提示される

### Edge Cases

- 空文書または空行のみの文書を保存した場合でも、利用者が意図した空状態として保存できるか
- 1MB 程度の文書で編集後に保存した場合でも、操作待ちが長時間にならないか
- 改行コードが混在している既存文書で保存した場合、既存文書の改行形式と末尾改行有無を維持できるか
- 編集開始後に同一文書が外部で変更されていても、本 Story では検知せず上書き保存する方針を利用者に明示できるか
- 未対応要素が複数箇所にある場合、警告が「保存で変化する可能性がある」ことを過不足なく伝えられるか
- 保存失敗時に「保存対象不正」「対象文書を保存不能」「一時的失敗」が判別でき、理由と再試行可否を提示できるか

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: システムは、閲覧中の Markdown 文書に対して編集モードへの切り替え手段を提供しなければならない。
- **FR-002**: システムは、編集モード開始時に対象文書の現在本文を編集画面へ表示しなければならない。
- **FR-003**: システムは、本文、見出し1、見出し2、見出し3、太字、イタリックを編集画面上で設定できなければならない。
- **FR-004**: システムは、行頭入力 `# `、`## `、`### ` をそれぞれ見出し1〜3として解釈しなければならない。
- **FR-005**: システムは、太字・イタリックに対応する入力ルールを編集画面で解釈し、対応する書式として表示しなければならない。
- **FR-006**: システムは、編集内容を Markdown 形式へ変換し、元の Markdown 文書へ保存できなければならない。
- **FR-007**: システムは、見出し1〜3を保存時に `#`、`##`、`###` として表現しなければならない。
- **FR-008**: システムは、保存成功後に利用者が最新内容を閲覧できるよう表示を更新しなければならない。
- **FR-009**: 利用者は、保存せずに編集を破棄して閲覧画面へ戻る操作を実行できなければならない。
- **FR-010**: システムは、未保存変更がある状態で別ファイル選択、編集終了、ブラウザ離脱を行う際に確認を表示しなければならない。
- **FR-011**: システムは、編集内容をページ再読込後に復元する機能を提供しない（MVP 範囲外）ことを明示しなければならない。
- **FR-012**: システムは、保存時に未対応要素の変換または欠損可能性がある場合、保存前に警告を表示し、利用者が保存継続またはキャンセルを選択できなければならない。
- **FR-020**: システムは、未対応要素を含む文書を保存する際、未対応要素の原文を可能な限りそのまま保持して保存しなければならない。
- **FR-021**: システムは、本 Story の保存処理において外部変更の検知（更新時刻比較、内容ハッシュ比較）を行わず、保存操作時は現在編集内容で上書き保存しなければならない。
- **FR-013**: システムは、保存対象を解決済みルート配下の Markdown 文書に限定しなければならない。
- **FR-014**: システムは、拡張子 `.md` 以外のファイルへの保存要求を拒否しなければならない。
- **FR-015**: システムは、対象が include 条件を満たし exclude 条件に該当しない場合にのみ保存を許可しなければならない。
- **FR-016**: システムは、`..` 等を用いたパス移動を含む不正な保存要求を拒否しなければならない。
- **FR-017**: システムは、UTF-8 を前提に保存し、既存文書の改行コードと末尾改行有無を維持しなければならない。
- **FR-018**: システムは、保存成功と保存失敗を利用者が判別できる形で提示しなければならない。保存失敗は少なくとも「保存対象の検証不正」「対象文書の不存在または権限不足等による保存不能」「一時的なI/O失敗」に分類し、各分類で理由と再試行可否を提示しなければならない。
- **FR-019**: システムは、1MB 程度の Markdown 文書に対して編集開始から保存結果表示までを実用上問題ない待ち時間で完了しなければならない。

### Key Entities _(include if feature involves data)_

- **編集セッション**: 利用者が1つの文書を編集中に保持する作業状態。対象文書識別子、現在の編集内容、未保存変更の有無を持つ。
- **文書内容スナップショット**: 編集開始時点または保存時点の本文状態。比較や破棄判断に使用される。
- **保存リクエスト**: 利用者が保存操作で確定した入力。保存先識別子、変換後 Markdown、検証結果（保存可否理由を含む）を持つ。
- **保存結果通知**: 保存処理の成功または失敗を利用者へ伝える結果情報。結果区分、メッセージ、再操作可否の案内を持つ。

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: テスト参加者の 90% 以上が、初見で 5 分以内に「編集開始から保存完了」までを完遂できる。
- **SC-002**: 見出し1〜3、太字、イタリックを含む検証用文書の保存後比較で、対象要素の意味保持率が 100% である。
- **SC-003**: 未保存変更ありの離脱系操作（別ファイル選択、編集終了、離脱）で、確認表示の出現率が 100% である。
- **SC-004**: 保存可否判定が必要な不正保存ケース（ルート外、非 Markdown、除外対象、パス移動）で、誤って保存される件数が 0 件である。
- **SC-005**: 1MB 程度の検証文書において、編集開始から保存結果表示までの完了時間が 95% の試行で 3 秒以内である。

## Assumptions

- 本 Story の対象利用者は閲覧機能を使えており、編集対象文書へアクセスできる前提とする。
- 本 Story の対象書式は本文、見出し1〜3、太字、イタリックに限定し、下線は対象外とする。
- 未対応要素は保存前警告の対象とし、保存時は原文パススルーを優先するが、保持不能な場合は内容変化の可能性を利用者へ提示する。
- 更新時刻比較、内容ハッシュ比較、競合警告、保存拒否を伴う競合検知は Story 011 の責務とし、本 Story では扱わない。
- エンコード不正およびサイズ超過は、本 Story では独立した失敗分類として扱わない。
- 保存後の表示更新方法は実装設計に委ねるが、利用者視点で最新表示が得られることを必須とする。
