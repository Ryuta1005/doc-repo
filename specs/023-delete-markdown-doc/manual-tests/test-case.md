## MT-001 ホバー時のみミートボールメニューが出ることを確認する

目的:

- サイドバー上のファイル行とフォルダ行で、ホバー時だけ行末にミートボールメニューが表示されることを確認する。

手順:

1. `node dist/cli/index.js serve` を起動する。
2. ブラウザで表示 URL を開く。
3. 任意のファイル行にマウスを置く。
4. 行末にミートボールメニューが表示されることを確認する。
5. マウスを外してメニューが消えることを確認する。
6. 任意のフォルダ行でも同様に確認する。
7. ホバーしていない別の行に常時メニューが出ていないことを確認する。

期待結果:

- ホバーした行だけにミートボールメニューが表示される
- ファイル行とフォルダ行の両方で同じ導線が使える
- 非ホバー状態ではメニューが表示されない

## MT-002 削除メニュー項目と確認ダイアログの表示を確認する

目的:

- ミートボールメニューから削除項目を開いたときに、仕様どおりの文言とボタンが出ることを確認する。

手順:

1. 任意のファイル行のミートボールメニューを開く。
2. ゴミ箱アイコン付き `削除` が表示されることを確認する。
3. その項目をクリックする。
4. 確認ダイアログ内に対象名が識別可能な形で表示されることを確認する。
5. h4 相当の見出し `削除しますか？` が表示されることを確認する。
6. `本当に削除しますか？` が表示されないことを確認する。
7. `Git Commitなどをしていない場合、元に戻せません。` が表示されることを確認する。
8. 赤背景白文字の `削除` ボタンと、現在の未保存変更確認ダイアログと同じデザインの `キャンセル` ボタンがあることを確認する。
9. フォルダ行でも同様に確認する。

期待結果:

- メニュー項目にゴミ箱アイコンと `削除` が表示される
- 確認ダイアログに対象名と警告文が表示される
- `削除` と `キャンセル` が仕様どおりの見た目で表示される

## MT-003 単一 Markdown 文書の削除成功を確認する

目的:

- 単一ファイル削除が成功し、ツリー更新と選択状態の更新が行われることを確認する。

手順:

1. 削除対象の Markdown 文書を 1 件選ぶ。
2. 対象行のミートボールメニューから `削除` を開く。
3. 確認ダイアログで対象名を確認する。
4. `キャンセル` を押して一度閉じる。
5. 再度ダイアログを開く。
6. `削除` を押して確定する。
7. 対象文書が削除されることを確認する。
8. 文書ツリーから対象が消え、残っている項目へ自然に遷移するか、残存項目がない場合は空状態になることを確認する。

期待結果:

- 対象 Markdown 文書だけが削除される
- 削除後に文書ツリーが更新される
- 存在しない文書が選択状態として残らない

## MT-004 フォルダ削除成功と配下一括削除を確認する

## MT-006 不正・不適切な削除要求の拒否を確認する

- rootDir 外、`.md` 以外、include/exclude 条件外、path traversal、存在しない対象の削除が拒否されることを確認する。

2. `.md` 以外のファイルを削除しようとする。

5. すでに存在しないファイルまたはフォルダを削除しようとする。

## MT-007 未保存変更ガードが削除確認より先に出ることを確認する

- 編集中の未保存変更がある状態で削除操作を開始したとき、既存の未保存変更確認が先に表示されることを確認する。

6. 削除を確定した場合、削除が実行されることを確認する。

- 未保存変更確認が削除確認より先に出る

## MT-008 削除後の選択遷移、空状態、応答時間を確認する

手順:

1. 残存文書が 2 件以上ある状態を用意する。

7. 削除後に存在しない文書が選択状態として残らないことを確認する。

期待結果:

- 削除成功後に残存項目へ自然に遷移する、または空状態が表示される
- 存在しない項目が選択状態として残らない
- 反映は 2 秒以内に完了する
