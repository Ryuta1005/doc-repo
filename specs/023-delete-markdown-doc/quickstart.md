## Prerequisites

- Node.js 20 以上
- 依存関係インストール済み

```bash
npm install
npm run build
```

## 1. サーバー起動

```bash
node dist/cli/index.js serve
```

確認項目:

- ワークスペース画面が表示される
- サイドバーに文書ツリーが表示される

## 2. ミートボールメニュー表示確認

1. サイドバーのファイル行へホバーする
2. 行右側にミートボールメニューが表示されることを確認する
3. ホバー解除でミートボールメニューが消えることを確認する
4. フォルダ行へホバーして同様に確認する
5. 期待結果: 同時に複数行でメニューが常時表示されない

## 3. メニュー項目確認

1. ファイル行のミートボールメニューをクリックする
2. 期待結果: ゴミ箱アイコン付き `削除` 項目が表示される
3. フォルダ行でも同様に確認する
4. 期待結果: file/folder どちらも削除導線を持つ

## 4. ファイル削除確認

1. 削除対象の Markdown 文書を 1 件用意する
2. 対象ファイル行のミートボールメニューから `削除` を押す
3. 期待結果: 確認ダイアログに h4 相当の見出し `削除しますか？`、対象名、`Git Commitなどをしていない場合、元に戻せません。` が表示され、`本当に削除しますか？` は表示されない
4. `キャンセル` を押す
5. 期待結果: 削除されない
6. 再度ダイアログを開き、赤背景白文字の `削除` を押す
7. 期待結果: 対象ファイルが削除され、ツリーから消え、残存項目へ遷移する

## 5. フォルダ削除確認（Policy B 成功ケース）

1. 配下が管理対象 Markdown のみで構成されたフォルダを用意する
2. フォルダ行のミートボールメニューから `削除` を押す
3. 確認ダイアログで対象フォルダ名と、配下もまとめて削除されることを確認する
4. `削除` を押す
5. 期待結果: 配下 Markdown と対象フォルダ自体が削除され、空ディレクトリ残骸が残らない

## 6. フォルダ削除拒否確認（Policy B 失敗ケース）

1. フォルダ配下に画像、PDF、設定ファイルなど管理対象外ファイルを 1 件以上配置する
2. フォルダ削除を実行する
3. 期待結果: フォルダ全体の削除が拒否される
4. 期待結果: 失敗理由が表示され、配下ファイルは一切削除されない

## 7. スコープ/安全性確認

- rootDir 外解決を試みる target: 拒否される
- `.md` 以外の file target: 拒否される
- include/exclude 条件外 target: 拒否される
- 存在しない file/folder target: 失敗理由が表示される
- `..` を含む path traversal 相当 target: 拒否される

## 8. 未保存変更ガード確認

1. 既存文書を編集して未保存状態にする
2. サイドバーのミートボールメニューから `削除` を押す
3. 期待結果: 既存の未保存変更確認が先に表示される
4. `キャンセル` 相当を選ぶ
5. 期待結果: 削除確認へ進まない
6. 再度実施し、破棄側を選ぶ
7. 期待結果: 削除確認へ進み、その後 delete を継続できる

## 9. SC-004 応答時間確認（2秒以内）

1. 削除対象を 1 件選ぶ
2. 確認ダイアログで `削除` を押した時刻を `t0` とする
3. ツリー反映と次選択または空状態表示が完了した時刻を `t1` とする
4. `t1 - t0` を 10 回計測し、p95 が 2 秒以内であることを確認する
5. 超過ケースは対象件数、folder/file 種別、配下件数とともに記録する

## 10. 推奨検証コマンド

## 11. 検証結果の記録ガイド

- 各シナリオで「操作」「期待結果」「実結果」「差分」を記録する
- 失敗時は target 種別、resolved path、include/exclude 条件、配下 unmanaged entry の有無を残す
- folder delete 失敗は、どの entry が拒否原因だったかを明記する
