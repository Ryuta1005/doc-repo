## 価値

閲覧体験を維持したまま、ブラウザ上で Confluence のようなリッチテキスト編集を行い、リポジトリ上の正本は Markdown として保存できる最小のワークフローを成立させる。まず動く価値を1本通すことで、後続の安全化や便利機能を段階的に積み上げられる。

## 受け入れ条件の要点

- Task 019 が Done である
- Task 020 が Done である
- Story 010 が Done である
- Phase 2 までの閲覧・serve・watch 体験を React + Hono 境界で再現できる
- doc-repo の正規入口が `serve` ベースの React + Hono ワークスペースとして整理されている
- ツールバーから本文、見出し1〜3、太字、イタリックを設定できる
- 対応済みの Markdown 入力ルールを直接入力すると、編集画面上でリッチテキストへ反映される
- ブラウザ上の編集内容を Markdown として元ファイルへ保存できる
- 保存成功後、生成済み表示が自動的に更新される
- 保存対象の基本制約を満たし、rootDir 外や `.md` 以外への保存を拒否できる
