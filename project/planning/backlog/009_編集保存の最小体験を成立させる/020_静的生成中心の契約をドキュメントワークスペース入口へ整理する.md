## 目的

doc-repo の中心価値を「static documentation site generator」から、Git と Markdown を正本にしたブラウザベースのドキュメントワークスペースへ整理する。020 では静的生成を即時廃止方針とし、プロダクト説明・CLI 契約・テスト・ロードマップ上の正規入口を `serve` ベースの React + Hono ワークスペースへ統一する。

## 受け入れ条件

- プロダクト説明で、doc-repo を静的サイト生成ツールではなく、Markdown ドキュメントをブラウザ上で閲覧・編集するドキュメント管理ツールとして表現できている
- `serve` が React Viewer の正規入口であることを README、product、roadmap、architecture、backlog のいずれか必要箇所へ反映できている
- 設定ファイルや CLI の `port` は Front/Back に分かれず、Hono が React UI・HTTP API・SSE・静的アセットを同一 port / 同一 origin で配信する前提になっている
- `doc-repo [scopePath]` による静的生成は廃止方針として明記され、正規導線から除外されている
- 静的生成廃止に伴う既存テスト・旧テンプレート UI の扱い（削除）が Spec/research/実装で一致している
- 旧テンプレートUI（`templates/page.html`, `templates/styles.css`, `templates/app.js`）が削除済みである
- 静的生成中心のテストが削除対象として整理され、不要な回帰テストが残っていない
- 010 の編集・保存機能が、静的生成前提ではなく `serve` の React + Hono ワークスペース前提で設計開始できる

## 技術検証

- 静的生成を完全廃止する場合の影響範囲を確認する
  - README/package description
  - CLI contract
  - `doc-repo [scopePath]` の既存テスト
  - `templates/` と `core/site/renderPages`
  - オフライン閲覧に関する既存仕様
- 019 の完了条件を壊さず、010 の保存 API / 編集 UI 設計に不要な静的生成前提が入り込まないことを確認する
