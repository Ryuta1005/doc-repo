# 利用者はリッチテキストエディタで文書を編集してMarkdownとして保存できる。なぜならMarkdown記法を知らなくても、その場でドキュメントを修正したいから。

- Spec: -
- Depends On: 019

## ユーザー価値

利用者は閲覧中の文書を Confluence のようなリッチテキストエディタで編集し、元の Markdown ファイルへ保存できる。Markdown を知らない利用者はツールバーで編集でき、Markdown を知っている利用者は対応済み記法を直接入力して同じ文書構造へ変換できる。

## 受け入れ条件

- 閲覧画面から編集モードへ切り替えられる
- 選択中の Markdown 本文がリッチテキスト編集画面へ表示される
- ツールバーから本文、見出し1〜3、太字、イタリックを設定できる
- 見出し1〜3は Markdown 保存時に `#`, `##`, `###` として表現される
- 対応済みの Markdown ショートカットを直接入力すると、編集画面上でリッチテキストへ反映される
- MVP で扱う Markdown 入力ルールは、行頭の `# `, `## `, `### ` と、太字・イタリックに対応する入力ルールを最低範囲とする
- 下線は MVP では扱わず、別 Story または Spike 後の判断対象とする
- 編集中の内容はブラウザのメモリ上で保持し、ページ再読込後の復元は MVP では扱わない
- 編集内容を Markdown として元の Markdown ファイルへ保存できる
- 保存成功後、生成済み表示が自動的に更新される
- 編集を破棄して閲覧画面へ戻れる
- 未保存変更がある状態で別ファイル選択、編集終了、ブラウザ離脱を行う場合は確認できる
- MVP 対応要素は意味と構造を維持して Markdown へ保存できる
- 未対応要素を含み、保存によって変換または欠損する可能性がある場合は、保存前に警告してキャンセルできる
- 未対応要素の原文保持範囲は Spike 結果に基づき Spec で定義する
- 保存対象は解決済み rootDir 配下の Markdown に限定される
- `.md` 以外のファイルへの保存を拒否する
- include 対象かつ exclude 対象外のファイルのみ保存できる
- `..` 等によるパストラバーサルを拒否する
- UTF-8 前提で保存し、既存の改行コードと末尾改行の有無を維持できる
- 1MB 程度の Markdown で編集・保存が体感上問題なく完了する
- 保存成功・保存失敗を利用者が確認できる

## 技術検証

- React や WYSIWYG エディタライブラリの採否は実装時の Spec または research 文書で Spike 結果を記録する
- Spike では、既存 Markdown を読み込み、無編集で保存しても意味や未対応部分が壊れないことと、ツールバー操作が普通のエディタとして十分使いやすいことを確認する
- 保存 API は必要とし、path、content、保存結果、代表エラーを返す契約を Spec で定義する。競合検知のための版情報照合は Story 011 の責務とする
- 保存成功後の表示更新は必須とするが、保存 API が生成処理を直接呼ぶか、watch 機構や部分再生成を使うかは Spec または設計で決定する
