# 利用者は他の変更を誤って上書きせずに保存できる。なぜなら安全にファイルを更新したいから。

- Epic: 016

- 優先度: P2
- Spec: -
- Depends On: 010

## ユーザー価値

編集中に VS Code や Git 操作などで発生した外部更新との競合を検知し、誤上書きを防いだうえで保存判断を行える。

## 受け入れ条件

- Chokidar / SSE などで外部変更を検知し、編集画面へ通知できる
- 未保存変更がある場合、外部変更を検知しても本文を自動リロードしない
- 保存時にバージョン、更新時刻、ハッシュなどの突合で不一致を検知できる
- 不一致時は上書きを拒否し、競合として扱える
- 利用者が保存中止または再読込を選択できる
- 初期段階では自動マージや差分マージを行わない
