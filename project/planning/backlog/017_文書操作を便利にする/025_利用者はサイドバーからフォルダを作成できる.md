## ユーザー価値

サイドバーの作成導線からファイルだけでなくフォルダも作成できることで、利用者はエディタから離れずに文書の分類先を整えられる。

## 受け入れ条件

- サイドバー下部の `+ 作成` とフォルダ行ホバー時の `+ 作成` をクリックすると、直接新規ファイル作成画面へ遷移せず作成種別メニューが表示される
- 作成種別メニューには `ファイル` と `フォルダ` の 2 候補が表示される
- `ファイル` を選択すると、既存の新規 Markdown 文書作成フローと同じ挙動になる
- `フォルダ` を選択すると、サイドバー内の追加されるべき位置にフォルダ名入力用のプレースホルダーが表示される
- プレースホルダーで有効なフォルダ名を入力して Enter を押すと、対象位置にフォルダが作成される
- フォルダ行起点では当該フォルダ直下にフォルダが作成される
- サイドバー下部の作成導線ではルート直下にフォルダが作成される
- 作成された空フォルダはサイドバー再読込後も表示される
- フォルダ名に `/` `\` `.` `..` などパス区切りまたはパスセグメントとして不適切な入力を指定した場合は作成されず、理由が表示される
- 同名のファイルまたはフォルダが既に存在する場合は上書きせず、重複エラーが表示される

## 実装上の前提

- 現在の文書ツリーは Markdown ファイル一覧から組み立てているため、空フォルダを永続表示するには一覧 API またはツリー用データモデルでフォルダノードを扱う必要がある
- フォルダ作成用 API を追加するか、既存の作成 API をファイル/フォルダの両方に対応できる形へ拡張する
- フォルダ名には `.md` を自動付与しない
- Story 022 の新規 Markdown 文書作成フローは既存挙動として維持する
