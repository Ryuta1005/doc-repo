# 利用者は表を挿入して行列を編集できる。なぜなら構造化された情報をMarkdown文書に整理したいから。

## メタ情報

- 種別: Story
- 番号: 024
- Epic: 017
- ステータス: Todo
- 優先度: P3
- Spec: -
- Depends On: 010

## ユーザー価値

表をリッチテキストエディタ上で追加・編集できることで、Markdown 記法を直接書かなくても比較表、一覧、整理表などの構造化された情報を文書に残せる。

## 受け入れ条件

- ツールバーの表ボタンから、n x n の行列グリッドを選択して表を挿入できる
- グリッド上で選択した行数・列数に応じた Markdown 表が文書へ挿入される
- 挿入された表はリッチテキストエディタ上でセル内容を編集できる
- 表の列境界または列操作 UI から、列を追加・削除できる
- 表の行境界または行操作 UI から、行を追加・削除できる
- 既存の Markdown 表を読み込んだ場合も、表として編集できる
- 保存後に再表示しても、表の行数・列数・セル内容が保持される
- 表以外の本文内容や未対応 Markdown 要素を破壊しない

## 実装上の前提

- Tiptap の table 系 extension を利用して表編集の土台を作る
- Markdown の parse/serialize 中間表現に table, tableRow, tableCell 相当のノードを追加する
- OneNote や Confluence に近い操作感は目指すが、詳細な UI 仕様は Speckit のフローで決める

## 実装タスク管理

- 機能実装タスクは Speckit のフローで管理する
