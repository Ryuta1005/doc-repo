# 利用者は不要なMarkdown文書を削除できる。なぜならブラウザ上でドキュメントを整理したいから。

## メタ情報

- 種別: Story
- 番号: 023
- Epic: 021
- ステータス: Todo
- 優先度: P2
- Spec: -
- Depends On: 010

## ユーザー価値

利用者は不要になった Markdown 文書をワークスペース上で削除し、文書一覧を整理できる。

## 受け入れ条件

- 文書ツリーまたはワークスペース上の操作から選択中の Markdown 文書を削除できる
- 削除前に対象ファイル名を含む確認を表示できる
- 削除対象は rootDir 配下の `.md` ファイルに限定される
- `..` 等によるパストラバーサルを拒否できる
- include 対象かつ exclude 対象外のファイルのみ削除できる
- 対象ファイルが存在しない場合は失敗理由を表示できる
- 削除成功後、文書ツリーが更新され、残っている文書へ自然に遷移できる
- 未保存変更がある状態で削除操作を行う場合、既存の未保存変更確認を尊重する
- 初期段階ではゴミ箱・復元機能は扱わず、Git で復元できる前提とする

## 実装タスク管理

- 機能実装タスクは Speckit のフローで管理する
