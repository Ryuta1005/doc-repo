# 利用者は新しいMarkdown文書を作成できる。なぜならブラウザ上でドキュメントを増やしたいから。

## メタ情報

- 種別: Story
- 番号: 022
- Epic: 021
- ステータス: Todo
- 優先度: P2
- Spec: -
- Depends On: 010

## ユーザー価値

利用者はエディタ外のファイル操作をせずに、新しい Markdown 文書を作成してすぐ編集できる。

## 受け入れ条件

- 文書ツリーまたはワークスペース上の操作から新しい Markdown 文書を作成できる
- 作成先パスまたはファイル名を入力できる
- 作成対象は rootDir 配下の `.md` ファイルに限定される
- `..` 等によるパストラバーサルを拒否できる
- include 対象かつ exclude 対象外のファイルのみ作成できる
- 既に同名ファイルが存在する場合は上書きせず失敗理由を表示できる
- 作成成功後、文書ツリーが更新され、作成した文書を選択できる
- 初期本文は空または最小見出しなど、Spec 作成時に決めたテンプレートで作成される
- 未保存変更がある状態で作成操作を行う場合、既存の未保存変更確認を尊重する

## 実装タスク管理

- 機能実装タスクは Speckit のフローで管理する
