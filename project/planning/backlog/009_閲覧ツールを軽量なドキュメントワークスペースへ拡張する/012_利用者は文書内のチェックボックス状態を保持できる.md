# 利用者は文書内のチェックボックス状態を保持できる。なぜなら進捗管理に利用したいから。

## メタ情報

- 種別: Story
- 番号: 012
- Epic: 009
- ステータス: Todo
- 優先度: P3
- Spec: -
- Depends On: 010

## ユーザー価値

チェックボックスの更新結果を維持できることで、Markdown 編集・保存の中核機能に追加して進捗管理価値を提供できる。

## 受け入れ条件

- 文書内チェックボックスの変更を保持できる
- 再表示後も保持された状態を確認できる
- 保持対象が文書単位で識別される
- 保持処理で他の本文内容を破壊しない
- Phase 3 の最小 Done 条件からは除外し、後半スライスまたは次 Phase 候補として扱う

## 実装タスク管理

- 機能実装タスクは Speckit のフローで管理する
