# 002 生成者はワンコマンドで、閲覧者に共有できるドキュメントサイトを生成できる。なぜならリポジトリMarkdownをすぐに読める形で届けたいから。

- Depends On: -

## ユーザー価値

ワンコマンドで閲覧可能な成果物を配布できる。

## 受け入れ条件の要点

- 対象リポジトリで CLI を1回実行すると、リポジトリ直下に `.doc-repo` が生成される
- 実行時に `.md` を再帰探索し、`node_modules`、`.git`、`.doc-repo`、`dist` は収集対象外になる
- 収集した Markdown は HTML に変換され、`content.json` と閲覧ページ（`index.html`）として出力される
- 出力先はアトミック置換され、再実行時に古い生成物との不整合が残りにくい構成になっている
- 生成物（`index.html`、`app.js`、`styles.css`、`content.json`）だけでブラウザ閲覧が成立する
- 左ペインにフォルダ/ファイルツリーが表示され、フォルダの開閉ができる
- ファイル選択で右ペイン本文が切り替わり、選択中ファイルが視覚的に判別できる
- 文書選択は URL ハッシュ（`#doc=`）に同期され、リロード後も同じ文書を表示できる
- Markdown が0件でも異常終了せず、警告付きで成功終了（exit code 0）する
- 実行結果の成否は標準出力メッセージと終了コードで判断できる

## 実装メモ

- CLI 実装を追加（`node dist/cli/index.js` で実行可能）
- 出力先 `.doc-repo` のアトミック置換を実装
- Markdown 0件時の警告付き成功（exit code 0）を実装
- 失敗時のガイダンス表示と非0終了コードを実装
