# 003 生成者は生成する範囲を引数で指定できる。なぜならリポジトリの中で閲覧する範囲を絞りたいから。

## メタ情報

- 種別: Story
- 番号: 003
- Epic: 001
- ステータス: Done
- 優先度: P1
- Spec: A, B, C
- Depends On: 002

## ユーザー価値

必要な範囲だけを絞って生成できる。

## 仕様メモ

- 指定方法は位置引数 1 つとする
- 指定できる対象は 1 つのみとする
- パス解決はリポジトリルート基準とする
- 引数未指定時は従来どおりリポジトリ全体を生成する
- 無効なパス指定時はエラー終了（exit code 1）とする
- 実行結果の標準出力には、実際に解決された生成対象パスを表示する

## 受け入れ条件

- 既存の生成コマンドに位置引数でフォルダパスを 1 つ指定できる
- 指定したパスはリポジトリルート基準で解決される
- 指定したフォルダ配下の Markdown のみを収集してサイトを生成できる
- 引数未指定時は、従来どおりリポジトリ全体を対象に生成できる
- 存在しないパス、フォルダでないパス、リポジトリ外を指すパスを指定した場合は、理由が分かるメッセージを表示して終了コード 1 で終了する
- 生成成功時の標準出力に、実際に生成対象として解決されたパスを表示できる
- 生成されたツリー表示と本文表示は、指定した範囲内の Markdown のみを対象とする

## 実装タスク管理

- 機能実装タスクは Speckit のフローで管理する
