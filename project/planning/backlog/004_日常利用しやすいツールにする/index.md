## 価値

Markdown を更新するたびに手動でコマンドを打ち直さず、ローカルサーバーと変更監視で継続的に閲覧できる。設定ファイルで対象範囲を固定化し、雛形も生成できると、繰り返し使いやすいツールになる。

## 受け入れ条件の要点

- Story 005, 006, 007, 008 がすべて Done である
- `doc-repo serve` でローカルサーバーが起動し、ブラウザから閲覧できる
- Markdown を変更するとブラウザが自動更新される
- `doc-repo.config.json` で rootDir と include/exclude を設定できる
- `doc-repo init` で `doc-repo.config.json` の雛形を生成できる
