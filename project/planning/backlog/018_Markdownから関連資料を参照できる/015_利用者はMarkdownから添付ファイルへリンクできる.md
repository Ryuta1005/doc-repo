- Depends On: 014

## ユーザー価値

利用者は Markdown の通常リンクから PDF、CSV、ZIP、画像ファイルなどの添付ファイルへ移動できる。なぜなら、本文だけでなく関連資料やサンプルファイルもドキュメントサイト上の自然なリンクとして確認したいから。

## 受け入れ条件

- Markdown の通常リンクがリポジトリ内の非Markdownファイルを参照している場合、そのファイルを `serve` ワークスペースから開ける
- 添付ファイルは Hono の配信境界で扱い、`serve` 経由で参照できる
- 実在 Markdown ページへのリンクは Viewer 内の文書遷移として扱う
- 外部URL、ハッシュリンク、メールリンクなどはコピー対象にしない
- rootDir 外へ抜けるパスやパストラバーサルはコピー対象にしない
- 存在しない添付ファイル参照の扱い（警告・無視・失敗）は Spec 作成時に決定する
- 添付ファイルの MIME / Content-Type の扱いは最低限ブラウザで開く、またはダウンロードできる状態にする
- Story 014 の画像表示修正とは分離し、通常リンクで参照される非Markdownファイルを対象にする

## 技術検証

- 添付ファイル種別の対象範囲、存在しないファイル参照時の扱い、Content-Type の拡張要否は Spec または research 文書で整理する

## 実装タスク管理

- 機能実装タスクは Speckit のフローで管理する
