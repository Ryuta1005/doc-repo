# プロダクト概要

## プロダクト名

doc-repo

## 背景

近年は Spec Driven Development や AI-DLC の流れの中で、リポジトリ内に Markdown ドキュメントを大量に持つケースが増えている。
一方で、非エンジニアや開発に不慣れな利用者にとっては、Git クライアント・エディタ・Markdown 記法を前提に内容を確認する体験は依然として重い。

その結果、ドキュメントはリポジトリに存在していても、読む人のハードルが高く、共有資産として十分に活用されないことがある。

## 解決したい課題

- リポジトリ配下の Markdown を、エディタなしでブラウザから読みたい
- フォルダ構成を保ったまま、どこに何があるかを辿れるようにしたい
- 非エンジニアでも、`npm` コマンドだけで閲覧可能な成果物を得られるようにしたい

## 提供価値

- リポジトリ内ドキュメントを、ブラウザ閲覧向けの情報サイトとしてまとめて出力できる
- フォルダ構成と Markdown の対応関係を保つことで、元のリポジトリ構造を理解しやすくする
- Git やエディタの操作に不慣れな利用者にも、配布しやすい HTML として共有できる

## 想定ユーザー

- Markdown を多く含むリポジトリを管理する開発者
- 仕様書や議事録を Git 管理したいチーム
- リポジトリの中身をブラウザで確認したい非エンジニアの関係者

## プロダクトの基本コンセプト

このプロダクトは、リポジトリを Markdown の保存場所として維持しつつ、閲覧体験だけをブラウザ向けに最適化するための npm パッケージである。

利用者は開発対象リポジトリでパッケージをインストールし、`doc-repo` コマンドを実行するだけで、リポジトリ配下の Markdown 一覧と本文を閲覧できる静的 HTML サイトを生成できる。

実行すると静的サイトが生成され、`index.html` を開けば Jest のカバレッジレポートのように閲覧できる。

## MVP の定義

MVP の範囲は以下とする。

- リポジトリ配下の `.md` ファイルを再帰的に読み取れる
- フォルダ構成に基づいたナビゲーションを生成できる
- Markdown を HTML に変換して 1 つの静的サイトとして出力できる
- 生成物を `.doc-repo` ディレクトリ配下にまとめて出力できる
- 生成された HTML をブラウザでそのまま開いて閲覧できる
- 最低限の 2 ペイン UI を持つ
  - 左ペイン: フォルダ/ファイルのツリー
  - 右ペイン: 選択した Markdown の本文表示

## 非目標

MVP では以下は対象外とする。

- ブラウザ上での Markdown 編集
- ホットリロード付きローカル開発サーバー
- 設定ファイルによる include/exclude の詳細制御
- 全文検索やタグ管理などの高度な情報整理機能
- 認証やマルチユーザー機能

## 成功条件

- 初回利用者が、リポジトリで CLI を 1 回実行して HTML を生成できる
- 生成物の場所が `.doc-repo` に統一され、扱いに迷わない
- 非エンジニアに生成物を渡したとき、エディタなしでドキュメント閲覧が成立する
- リポジトリ構造と HTML 上のナビゲーション構造が対応している

## 将来像

将来的には、静的 HTML 生成だけでなく、ローカルサーバー起動・ホットリロード・ブラウザ編集から Markdown への反映まで段階的に拡張したい。

ただし初期設計では、MVP を静的サイト生成に限定しつつ、将来サーバー機能を追加しても中核ロジックを再利用できる構成を目指す。
