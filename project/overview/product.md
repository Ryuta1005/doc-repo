# プロダクト概要

## プロダクト名

doc-repo

## 背景

近年は Spec Driven Development や AI-DLC の流れの中で、リポジトリ内に Markdown ドキュメントを大量に持つケースが増えている。
一方で、非エンジニアや開発に不慣れな利用者にとっては、Git クライアント・エディタ・Markdown 記法を前提に内容を確認・修正する体験は依然として重い。

その結果、ドキュメントはリポジトリに存在していても、読む人のハードルが高く、共有資産として十分に活用されないことがある。

## 解決したい課題

- リポジトリ配下の Markdown を、エディタなしでブラウザから閲覧・編集したい
- フォルダ構成を保ったまま、どこに何があるかを辿れるようにしたい
- 非エンジニアでも、VS Code や Git を直接操作せずに内容を探したり修正したりできるようにしたい
- Git と Markdown を正本として維持しながら、OneNote や Confluence のような操作感で扱えるようにしたい

## 提供価値

- リポジトリ内ドキュメントを、ブラウザで閲覧・編集できるドキュメントワークスペースとして扱える
- フォルダ構成と Markdown の対応関係を保つことで、元のリポジトリ構造を理解しやすくする
- Git や Markdown を正本として維持したまま、職種を問わずドキュメントを利用できる
- ブラウザ上で編集した内容を元の Markdown へ保存できるため、Git 操作に不慣れな利用者もドキュメント更新に参加できる

## 想定ユーザー

- Markdown を多く含むリポジトリを管理する開発者
- 仕様書や議事録を Git 管理したいチーム
- リポジトリのドキュメントをブラウザで確認・修正したい非エンジニアの関係者

## プロダクトの基本コンセプト

このプロダクトは、リポジトリを Markdown の保存場所として維持しつつ、ブラウザから閲覧・編集できるドキュメント管理ツールである。

利用者は開発対象リポジトリでパッケージをインストールし、`doc-repo serve` を実行するだけで、リポジトリ配下の Markdown 一覧と本文をブラウザで閲覧・編集できる。ワークスペース起動中は Markdown の変更を監視し、更新をブラウザ表示へ反映できる。

ブラウザ上で編集した内容は元の Markdown ファイルへ保存されるため、非エンジニアでも OneNote や Confluence を操作するような感覚で Git リポジトリ上のドキュメントを扱える。

## MVP の定義

現在の最小提供範囲は以下とする。

- リポジトリ配下の `.md` ファイルを再帰的に読み取れる
- フォルダ構成に基づいたナビゲーションを表示できる
- Markdown を HTML に変換し、Hono API 経由で React Viewer に表示できる
- `doc-repo serve` で React UI、HTTP API、SSE、静的アセットを同一 origin で配信できる
- 最低限の 2 ペイン UI を持つ
  - 左ペイン: フォルダ/ファイルのツリー
  - 右ペイン: 選択した Markdown の本文表示

## 非目標

MVP では以下は対象外とする。

- 認証やマルチユーザー機能
- 全文検索やタグ管理などの高度な情報整理機能

## 成功条件

- 初回利用者が、リポジトリで CLI を 1 回実行してドキュメントワークスペースを起動できる
- リポジトリ内 Markdown の一覧と本文を、エディタなしでブラウザから閲覧できる
- ブラウザ上で編集した内容を、元の Markdown ファイルへ保存できる
- 非エンジニアでも、Git やエディタを直接操作せずにドキュメントの確認・修正ができる
- リポジトリ構造とブラウザ上のナビゲーション構造が対応している

## 段階的な拡張方針

doc-repo は、`serve` ベースのドキュメントワークスペースを正規入口として、閲覧、ホットリロード、ブラウザ上のリッチテキスト編集、Markdown への保存を段階的に積み上げる。

020 以降、静的生成の直接入口は廃止済みとし、サーバー機能や編集機能を追加しても application/core の中核ロジックを再利用できる構成を維持する。
